#### 4. Main debugging focus: RViz map display / Nav2 integration
A large part of today was spent debugging why the saved map could not be displayed correctly in RViz.

Key things I checked / learned:
- Fixed Frame must be `map`
- RViz display items need to be added correctly:
  - Map
  - TF
  - RobotModel
  - LaserScan
- `/map` existing does not automatically mean RViz will display it
- QoS and timing matter a lot in ROS2
- localization and navigation must be started in the right order
- simulation time (`/clock`, `use_sim_time`) is critical

After multiple rounds of troubleshooting, I finally got the saved black-and-white map to appear in RViz.
