## Key Understanding

### 1. Planner vs Controller
The planner determines where the robot should go, while the controller determines how the robot should move at each moment.


planner_server → global path
controller_server → cmd_vel


### 2. Global Costmap vs Local Costmap
Nav2 uses two costmaps with different responsibilities.


global_costmap → used by planner
local_costmap → used by controller


The global costmap represents the full environment, while the local costmap focuses on nearby obstacles and updates more frequently.

### 3. Inflation Layer
The inflation layer expands obstacle cost around walls and objects.


inflation_radius


If this parameter is too small, the robot may drive too close to obstacles.

### 4. DWB Local Planner
The DWB controller samples many candidate trajectories and evaluates them using multiple scoring criteria.

Examples of trajectory critics include:


BaseObstacle
PathAlign
PathDist
GoalAlign
GoalDist
RotateToGoal
Oscillation


The trajectory with the best score is selected and converted into a velocity command.
