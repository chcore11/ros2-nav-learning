# 2026-03-08 (Day9)

## Goal

Test navigation behavior and verify system stability after the Navigation2 stack was successfully deployed in simulation.

## What I did

- Started the simulation environment using the project startup script
- Launched Gazebo and the Navigation2 stack
- Initialized localization using **2D Pose Estimate** in RViz
- Performed multiple navigation tests using **Nav2 Goal**

Test scenarios included:

- Short-distance navigation
- Navigation around obstacles (pillars)
- Diagonal goal navigation
- Return navigation (A → B → A)

## Result

The navigation system operated stably during all tests.

Observed behaviors:

- Short-distance navigation worked reliably.
- When navigating around pillars, the robot occasionally hesitated slightly before selecting a path, but did not take excessively long routes.
- Diagonal navigation did not show the previous path oscillation behavior observed earlier.
- Return navigation remained stable, and the robot successfully reached the original position.

Overall system behavior remained consistent between Gazebo and RViz, indicating stable localization and navigation.

## Conclusion

The Navigation2 system demonstrates stable goal-based navigation in the simulation environment.

The robot can reliably reach target positions under different navigation scenarios, including obstacle avoidance and return navigation.

The navigation pipeline is confirmed to be working properly:

map_server → AMCL → planner_server → controller_server → cmd_vel → robot motion
