## Key Understanding

### 1. AMCL
AMCL is not used for mapping.
It estimates the robot pose on a known map using laser scan and odometry.

### 2. planner_server
The planner server computes a global path from the current pose to the goal.
It provides the route direction, not the low-level motion command.

### 3. controller_server
The controller server follows the global path and generates real-time velocity commands.
This explains why the robot may rotate first, hesitate slightly, or behave cautiously near obstacles.

### 4. costmap
The costmap represents free space, occupied space, and inflated obstacle cost.
It influences both path selection and local motion behavior.

### 5. bt_navigator
The bt_navigator acts as the high-level task coordinator.
It receives the goal and organizes planning, control, and recovery behaviors.

## Pipeline
RViz Goal
→ bt_navigator
→ planner_server
→ global path
→ controller_server
→ /cmd_vel
→ robot motion

Supporting modules:
map
→ AMCL localization
→ costmap updates
→ safer navigation behavior
