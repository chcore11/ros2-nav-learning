# 2026-03-05 (Day7) Nav2 Autonomous Navigation (AMCL + Nav2)

## What I did
- Launched Gazebo (`turtlebot3_world`)
- Started Nav2 with the saved map from Day6 (map_server + AMCL + planner/controller/bt_navigator)
- Opened RViz2 and loaded the TurtleBot3 Nav2 RViz config
- Set the initial pose using **2D Pose Estimate**
- Sent navigation goals using **Nav2 Goal** (robot can navigate to the clicked target)

## Issues / Fix
- RViz error: `Frame [map] does not exist` / map not displayed
  - Cause: `map_server` failed to load the map due to a wrong `image:` entry in the map YAML
  - Fix: corrected `image:` to the actual `.pgm` file name and restarted Nav2
  - Verification:
    - Topics exist: `/map`, `/tf`, `/tf_static`
    - Nodes exist: `/map_server`, `/amcl`, `/planner_server`, `/controller_server`, `/bt_navigator`

- Robot did not move (no `/cmd_vel`)
  - Cause: used **2D Goal Pose** (generic RViz tool) instead of **Nav2 Goal** (Nav2 action)
  - Fix: loaded the official TB3 Nav2 RViz config and used **Nav2 Goal**
  - Verification: `/cmd_vel` starts publishing when a goal is sent

- Navigation works but is not accurate yet (pose mismatch between map view and robot)
  - Likely causes: initial pose not precise / AMCL not fully converged / parameter tuning needed
  - Next step: check laser-to-map alignment and refine AMCL + costmap parameters
