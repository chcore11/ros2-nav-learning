# Day13 Log

## Goal

The goal of Day13 was to deeply understand the internal architecture of the ROS2 Nav2 navigation system.
Instead of only running the navigation stack, I focused on analyzing how the core components interact with each other, including the planner, controller, costmaps, and the Behavior Tree used by `bt_navigator`.

---

## What I Did

First, I inspected the navigation actions provided by Nav2.

Using ROS2 CLI tools:

ros2 action list
ros2 action info /navigate_to_pose

I confirmed that the navigation goal sent from RViz is handled by the `bt_navigator` node.

Then I checked which Behavior Tree configuration file is used by the navigator.

ros2 param get /bt_navigator default_bt_xml_filename

The result showed:

navigate_to_pose_w_replanning_and_recovery.xml

This file defines the navigation logic used by Nav2.

---

Next, I located the Behavior Tree files inside the Nav2 package:

/opt/ros/humble/share/nav2_bt_navigator/behavior_trees

After opening the main XML file:

navigate_to_pose_w_replanning_and_recovery.xml

I analyzed the navigation Behavior Tree structure.

Key nodes in the Behavior Tree include:

ComputePathToPose
FollowPath
ClearEntireCostmap
Spin
Wait
BackUp
GoalUpdated

These nodes correspond to actual navigation capabilities implemented in Nav2.

---

## Navigation Pipeline Understanding

From analyzing the Behavior Tree, I understood the main navigation pipeline:

RViz Goal
↓
navigate_to_pose action
↓
bt_navigator
↓
ComputePathToPose → planner_server
↓
FollowPath → controller_server
↓
cmd_vel
↓
Robot motion

Supporting modules:

AMCL → robot localization
Costmaps → environment representation
Laser scan → obstacle detection

The Behavior Tree coordinates these components and decides when each module should run.

---

## Costmap Observation

While observing the topic:

ros2 topic echo /local_costmap/costmap

I noticed that grid values increase as the robot gets closer to obstacles.

This behavior is caused by the **inflation layer**, which increases the cost around obstacles so that the robot keeps a safe distance instead of moving too close to walls or objects.

This confirmed how Nav2 represents environmental risk through costmaps.

---

## Recovery Behaviors

The Behavior Tree also defines several recovery strategies when navigation fails.

These include:

ClearEntireCostmap
Spin
Wait
BackUp

These recovery actions help the robot handle situations such as:

temporary obstacles
incorrect costmap data
local planner failure
blocked paths

The system will try these strategies before attempting to replan the path.
