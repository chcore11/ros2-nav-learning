## Key Understanding

### 1. Nav2 is a system, not a single program
Nav2 is composed of multiple ROS2 nodes working together.
Key nodes include:
- `amcl`
- `planner_server`
- `controller_server`
- `bt_navigator`
- `behavior_server`
- `map_server`
- `global_costmap`
- `local_costmap`

These nodes together form the navigation stack.

### 2. Launch entry point
The TurtleBot3 navigation system is started from:

`navigation2.launch.py`

This file acts as the entry point for navigation startup.
It loads the map, loads the parameter file, and then includes the bringup launch process.

### 3. Bringup process
The launch system then calls:

`bringup_launch.py`

This bringup stage is responsible for starting the main Nav2-related nodes and organizing the navigation runtime environment.

### 4. Lifecycle manager
Nav2 uses lifecycle nodes instead of ordinary nodes for its core components.

Typical lifecycle flow:

unconfigured
→ inactive
→ active

The lifecycle managers are responsible for controlling these transitions and ensuring that nodes start in the correct order.

### 5. Why lifecycle matters
A navigation system cannot start randomly.
For example, the controller should not begin operating before the planner, map server, and localization system are ready.

The lifecycle managers ensure that startup is ordered, stable, and controlled.

### 6. Runtime understanding
The runtime navigation pipeline can be summarized as:

RViz Goal
→ bt_navigator
→ planner_server
→ global path
→ controller_server
→ /cmd_vel
→ robot motion

Supporting modules include:
- `map_server` for the map
- `AMCL` for localization
- `global_costmap` for planning
- `local_costmap` for local obstacle avoidance
