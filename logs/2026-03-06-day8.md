## Goal
Verify the Navigation2 stack using the saved map and confirm that the robot can navigate to user-defined goals in the simulation environment.

## What I did
- Restarted the system from a clean environment
- Launched TurtleBot3 Gazebo simulation
- Started the Navigation2 stack using the saved map
- Verified required navigation nodes:
  - /map_server
  - /amcl
  - /planner_server
  - /controller_server
  - /bt_navigator
- Checked lifecycle states of navigation nodes
- Opened RViz2 with the TurtleBot3 Navigation2 configuration
- Initialized robot localization using **2D Pose Estimate**
- Sent navigation goals using **Nav2 Goal**
- Performed multiple navigation tests across the map
- Tested navigation around obstacles such as pillars
- Verified that the robot can reach different target positions

## Result
Successfully verified that the Navigation2 stack is working with the saved map.

Robot behavior:
- Robot can navigate to user-defined positions
- Path planning correctly avoids obstacles
- Navigation works after restarting the system
- Robot can move across the map and reach the specified goals

Observed behavior:
- When navigating to diagonal targets, the robot sometimes hesitates between two possible routes
- This happens when both paths have similar navigation cost
- The robot briefly evaluates both options before committing to one

Despite this behavior, the robot consistently reaches the final target.

## Navigation Pipeline Verification
The full navigation pipeline was confirmed working:

map_server → AMCL → TF (map → odom → base_link)  
→ Global Planner → Local Controller → cmd_vel  
→ Gazebo robot motion

The robot is now capable of goal-based autonomous navigation in the simulation environment.
